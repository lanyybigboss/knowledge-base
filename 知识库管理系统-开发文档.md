- [x] 文档解析（mammoth.js / pdf.js / tesseract.js）── 内置于各上传/详情组件

> #### 后台回扫机制
>
> | 参数 | 值 | 说明 |
> |------|-----|------|
> | 首次触发 | 启动后 **5 秒** | 给应用初始化留时间 |
> | 扫描间隔 | **30 秒** | 定时检查是否有新未分析文档 |
> | 处理策略 | **逐个串行** | 质量优先，一次只分析一个，完成后再取下一个 |
> | 跳过条件 | 已分析 / 内容<20字符 / 正在处理中 | 避免重复分析和无效文档 |
> | 超时控制 | **120 秒** | 单次分析超过 2 分钟自动终止，降级到空结果 |
> | 健康重试 | **30 秒** | 首次检测失败后每 30 秒重新探测 Ollama 服务 |
>
> **数据结构**：
> - `_pendingIds: Set` — 防止同一文档重复入队
> - `_running: boolean` — 防止多轮扫描并发
> - 分析完成后写入 `aiAnalyzed: true` 标记，不再回扫
>
> **手动触发**：`backgroundAnalysisService.scanNow()` 可即时扫描一次
>
> **状态查询**：`backgroundAnalysisService.getStatus()` → `{ running, processing, pending }`
>
> **AI 分析降级策略（v1.5.1）**：
> ```
> analyzeDocument(content, title, fileName)
>     │
>     ├── shouldUseOllama() → 检测 Ollama 健康状态
>     │     ├── 可用 → analyzeDocumentOllama()
>     │     │     ├── JSON 解析成功 → normalizeResult() → 返回
>     │     │     ├── JSON 解析失败 → normalizeResult(降级字段) → 返回 ← 不再穿透到 DeepSeek
>     │     │     └── 网络超时/抛异常 → 降级到 DeepSeek
>     │     └── 不可用 → 降级到 DeepSeek
>     │
>     └── DeepSeek
>           ├── API Key 可用 → 分析成功 → 返回
>           └── 无 API Key / 网络错误 → 返回降级结果(空摘要, 默认分类)
>
> 核心原则：analyzeDocument() 永不抛异常，始终返回有效结果对象
> ```
>
> **完整流程**：
> ```
> 应用启动 → 5秒后首次扫描 → 遍历全部文档 → 筛选(未分析 & 内容≥20字符 & 不在pending队列)
>     ↓ 逐个处理
> 取一个文档 → aiService.analyzeDocument() → Ollama优先 → DeepSeek降级
>     ↓ 总是返回结果（成功/降级）
> 更新文档字段 + aiAnalyzed=true → onDocAnalyzed回调 → dispatch(UPDATE_DOCUMENT)
>     ↓
> 处理下一个... → 全部处理完毕 → 30秒后下一轮扫描
> ```

## ⚠️ 注意事项

1. **Electron 文件协议**：使用 HashRouter，所有资源相对路径引用
2. **存储容量**：IndexedDB 约 50MB+（由浏览器自动管理），大文件使用 `data/` 目录
3. **文件路径编码**：Windows 中文路径确保 UTF-8
4. **chokidar**：文件夹监听需动态 import
5. **打包权限**：NSIS 安装包需管理员权限，便携版无需

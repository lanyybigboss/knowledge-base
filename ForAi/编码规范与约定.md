## 编码约定

### 1. 日志系统（强制）

**禁止** `console.log/error/warn/debug`

**必须使用** `logger` 单例：
```js
import logger from './logger'
logger.info('消息', optionalData)
logger.error('错误', optionalData)
logger.warn('警告', optionalData)
```

**豁免**：`logger.js` 自身、`electron/` 目录

### 2. AI 分析降级策略

`analyzeDocument()` **永不返回 null**

| 场景 | 返回值 |
|------|--------|
| 成功 | 分析结果对象 |
| 失败 | `{ _fallback: true, category: 'other', ... }` |

**完整降级链**：
```
Ollama (动态模型解析) → MiMo → DeepSeek API → fallback
```

### 3. Ollama 动态模型解析（v1.7.x 新特性）

**模型前缀优先级**：
```js
['qwen3.6', 'qwen3', 'qwen2.5', 'qwen2']
```

**新特性**：
- 自动从 Ollama 模型列表中解析最佳可用模型
- `/no_think` 指令（qwen3 新特性）
- `think: false` 参数
- 请求失败时清缓存，下次重新解析

```js
// 请求示例
{
  model: 'qwen3:8b',  // 动态解析的实际模型名
  messages: [...],
  format: 'json',
  think: false        // 禁用思维链
}
```

### 4. 服务层模式

所有服务导出单例，提供 `start()/stop()` 生命周期：

| 服务 | 作用 |
|------|------|
| `backgroundAnalysisService` | 后台 AI 分析 |
| `backgroundAnalysisService.forceScan()` | 强制触发扫描（不受待机限制） |
| `syncService` | 跨模式数据同步 |
| `taskQueueService` | 任务队列 |

### 5. 存储层

- **数据库**：IndexedDB `KnowledgeBaseDB`
- **表**：`documents`, `categories`, `kvStore`, **`todos`** (v1.7.x 新增)
- **迁移**：`storageService.init()` 从 localStorage 迁移

### 6. 后台分析参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 扫描间隔 | 30 秒 | |
| 首次触发延迟 | 5 秒 | |
| 跳过阈值 | < 20 字符 | |
| 分析超时 | **300 秒 (5分钟)** | 需大于子进程全链路总超时 |
| 每轮最多处理 | 3 个文档 | |

### 7. 跨模式同步

| 模式 | 通信方式 |
|------|---------|
| Vite | HTTP `/api/sync/*` |
| Electron | IPC `sync-write/read/timestamp` |

- 脏标志 + 2 秒节流推送
- 5 秒轮询拉取
- 时间戳比较决定方向

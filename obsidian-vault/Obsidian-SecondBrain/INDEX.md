# INDEX — 记忆路由地图

> 这是 AI 的"导航地图"。告诉它：什么任务该读什么，什么时候该写什么。

---

## 系统架构

```
Obsidian-第二大脑/
├── 01-AGENTS/          ← AI 行为规则（必读）
├── 02-记忆/            ← 跨项目长期记忆
├── 03-Handoff/         ← 项目级接力文件
├── 04-项目/            ← 具体项目资料
├── 05-素材/            ← 原始素材、参考资料
├── 06-模板/            ← 可复用模板
└── 99-Archive/         ← 归档内容
```

---

## 快速导航

### 按任务类型

| 任务类型 | 先读 | 再读 | 写入位置 |
|---------|------|------|---------|
| 新任务启动 | AGENTS.md | INDEX.md | 03-Handoff/ 或 04-项目/ |
| 继续已有项目 | AGENTS.md | 对应 Handoff 文件 | 更新同一 Handoff |
| 记录长期偏好 | AGENTS.md | 02-记忆/偏好风格/ | 02-记忆/偏好风格/ |
| 记录踩坑经验 | AGENTS.md | 02-记忆/踩过的坑/ | 02-记忆/踩过的坑/ |
| 整理参考资料 | AGENTS.md | 05-素材/ | 05-素材/ |
| 项目结束归档 | AGENTS.md | 对应 Handoff | 99-Archive/ |

### 按内容分类

#### 02-记忆/ — 长期记忆库

| 子文件夹 | 存放内容 | 示例 |
|---------|---------|------|
| 偏好风格/ | 稳定的个人偏好 | 写作风格、审美倾向、决策习惯 |
| 验证过的流程/ | 已验证的工作流 | SOP、检查清单、最佳实践 |
| 踩过的坑/ | 失败经验 | 错误决策、无效尝试、教训总结 |
| 关键决策/ | 重要判断记录 | 方案选择依据、取舍理由 |

#### 03-Handoff/ — 项目接力

| 文件命名 | 用途 |
|---------|------|
| `项目名-handoff.md` | 单个项目的 canonical handoff |
| 更新规则：覆盖，不新建 | 每次推进后更新同一文件 |

#### 04-项目/ — 项目资料

| 子文件夹 | 状态 |
|---------|------|
| 进行中/ | 当前活跃项目 |
| 归档/ | 已完成项目（定期移到 99-Archive/） |

## 记忆写入决策树

```
任务结束，要写入记忆吗？
│
├─ 是一次性问答？→ 不写
│
├─ 是临时草稿？→ 不写
│
├─ 满足三问标准？
│  ├─ 长期有用？
│  ├─ 以后会复用？
│  └─ 能被验证？
│     ├─ 三个都满足 → 写入 02-记忆/ 对应分类
│     └─ 任一不满足 → 不写
│
└─ 是项目续接？
   ├─ 项目还要继续 → 更新 03-Handoff/ 对应文件
   └─ 项目已结束 → 移到 99-Archive/
```

## 快捷指令索引

| 用户说 | AI 执行 |
|-------|--------|
| "记住这个" | 分析内容 → 分类 → 写入 02-记忆/ |
| "项目交接" | 读取当前状态 → 更新 Handoff → 确认下一步 |
| "整理素材" | 归类 → 去重 → 存入 05-素材/ |
| "归档" | 确认内容 → 移动到 99-Archive/ |
| "新任务" | 读取 AGENTS → 读取 INDEX → 创建 Handoff |

## 文件命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| Handoff 文件 | `项目名-handoff.md` | `内容策划-handoff.md` |
| 记忆文件 | `主题-类型.md` | `写作风格-偏好.md` |
| 项目文件 | `日期-项目名-类型.md` | `2026-06-09-产品发布-计划.md` |
| 素材文件 | `来源-主题.md` | `抖音-AI工作流-笔记.md` |

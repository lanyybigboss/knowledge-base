# 第二大脑使用指南

> 基于 BowieMkt "Codex + Obsidian 第二大脑"方案落地的个人实践系统。

---

## 一、系统概述

### 核心理念

**Obsidian 负责记住，Codex/SOLO 负责执行。**

不是让 AI 帮你记笔记，而是让 AI 能读懂你、接住你、继续帮你推进事情。

### 三层架构

```
第一层：AGENTS.md     → AI 行为规则（宪法）
第二层：02-记忆/       → 跨项目长期记忆
第三层：03-Handoff/    → 项目级接力文件
```

---

## 二、快速开始

### Step 1：导入到 Obsidian

1. 下载 `Obsidian-第二大脑` 文件夹
2. 将整个文件夹放入你的 Obsidian 库中
3. 在 Obsidian 中确认文件结构完整

### Step 2：个性化 AGENTS.md

打开 `01-AGENTS/AGENTS.md`，修改：

- **第 1 节**：你的专业领域、语言偏好
- **第 2 节**：调整 AI 的工作边界
- **第 7 节**：添加你常用的快捷指令

### Step 3：首次使用 SOLO

当你想让 SOLO 基于你的笔记工作时：

1. **上传相关文件**到 SOLO（当前需要手动传输）
2. **告诉 SOLO**："先读 AGENTS.md 和 INDEX.md"
3. **再给出具体任务**

## 三、日常使用场景

### 场景 1：开始一个新项目

**你的动作**：
1. 复制 `03-Handoff/项目接力模板.md`
2. 重命名为 `项目名-handoff.md`
3. 填写项目信息

**对 SOLO 说**：
```
我要开始一个新项目「XXX」，
先读 AGENTS.md 和 INDEX.md，
然后读取 03-Handoff/XXX-handoff.md，
帮我制定项目计划。
```

### 场景 2：记录长期偏好

**当你发现 AI 反复问同样的问题时**：

**对 SOLO 说**：
```
记住这个：我的写作风格偏好是 XXX
```

SOLO 会：
1. 分析内容类型
2. 写入 `02-记忆/偏好风格/` 对应文件
3. 下次自动应用

### 场景 3：项目续接

**隔了几天继续推进项目时**：

**对 SOLO 说**：
```
继续项目「XXX」，
先读 AGENTS.md，
再读 03-Handoff/XXX-handoff.md，
基于当前进度继续推进。
```

SOLO 会：
1. 读取 Handoff 了解当前状态
2. 基于上下文继续工作
3. 更新 Handoff 文件

### 场景 4：记录踩坑经验

**当你试错后发现某个方法不行**：

**对 SOLO 说**：
```
记住这个坑：XXX 方法在 YYY 场景下无效，因为 ZZZ
```

SOLO 会写入 `02-记忆/踩过的坑/`，下次自动避开。

### 场景 5：项目归档

**项目完成后**：

**对 SOLO 说**：
```
项目「XXX」已完成，帮我归档
```

SOLO 会：
1. 更新 Handoff 状态为"已完成"
2. 将相关文件移到 `99-Archive/`
3. 保留关键经验到 `02-记忆/`

## 四、与 SOLO 的配合流程

### 当前工作流（手动传输）

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Obsidian   │ ←── │   微信/云盘  │ ←── │    SOLO     │
│  (手机本地)  │     │  (传输工具)  │     │  (云端处理)  │
└─────────────┘     └─────────────┘     └─────────────┘
      ↑                                    ↓
      └──────────── 下载结果文件 ────────────┘
```

### 具体操作

| 方向 | 操作 |
|------|------|
| Obsidian → SOLO | 导出 `.md` 文件 → 微信传输 → SOLO 读取 |
| SOLO → Obsidian | SOLO 生成文件 → 下载 → 导入 Obsidian |

### 未来优化方向

- **GitHub 同步**：Obsidian Git 插件 + SOLO GitHub 技能
- **iCloud/OneDrive**：云盘共享文件夹
- **WebDAV**：直接同步（需配置）

## 六、记忆写入标准

### 三问原则

在让 SOLO 写入任何长期记忆前，先问自己：

1. **长期有用吗？** — 不是一次性信息
2. **以后会复用吗？** — 有再次调用的价值
3. **能被验证吗？** — 不是猜测或假设

**三个都满足，再写入。**

### 什么不该写入

- ❌ 一次性问答的结果
- ❌ 临时草稿、中间产物
- ❌ 情绪化总结
- ❌ 未经确认的猜测

### 什么应该写入

- ✅ 跨项目可用的偏好风格
- ✅ 已验证的有效流程
- ✅ 反复踩过的坑
- ✅ 重要决策及理由
- ✅ 项目 handoff（项目还要继续时）

## 八、进阶技巧

### 1. 双向链接网络

在 Obsidian 中使用 `[[笔记名]]` 建立连接：
- Handoff 文件链接到相关记忆
- 记忆文件链接到相关项目
- 形成知识网络，AI 能更好理解关联

### 2. 标签系统

统一标签规范：
- `#项目/进行中`
- `#记忆/偏好`
- `#素材/待整理`
- `#待验证`

### 3. 模板扩展

在 `06-模板/` 中添加你自己的模板：
- 会议记录模板
- 读书笔记模板
- 项目复盘模板

### 4. 与 xlsx/pdf 技能联动

| 场景 | 操作 |
|------|------|
| 项目数据整理 | SOLO xlsx 技能 → 生成表格 → 存入 `05-素材/` |
| 报告输出 | SOLO 读取笔记 → 生成 docx/pdf → 分享 |
| 资料提取 | 上传 pdf → SOLO 提取 → 写入 Obsidian 笔记 |

## 九、常见问题

**Q：为什么 AI 每次都要先读 AGENTS.md？**
> 确保 AI 了解你的偏好和边界，避免每次重复沟通。

**Q：Handoff 和项目文件有什么区别？**
> Handoff 是"接力棒"，记录进度和下一步；项目文件是具体资料。

**Q：记忆文件会不会越来越多？**
> 会，所以要定期整理。好的记忆是"越用越薄"——经常复用的留下，过时的归档。

**Q：手机和电脑怎么同步？**
> 目前推荐：Obsidian Sync、iCloud、或 GitHub。选择适合你的方案。
